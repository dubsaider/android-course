# Lesson 01 - Demo на Jetpack Compose

В этой папке находится демо первого урока: минимальное Android-приложение на Jetpack Compose с двумя экранами и навигацией между ними.

## Цель демо

Показать базовые практики, которые нужны для старта:

- создание UI через `setContent {}` в `MainActivity`;
- описание экранов как `@Composable` функций;
- переходы между экранами через Navigation Compose;
- простая работа с состоянием (`remember` + `mutableStateOf`).

## Что реализовано

- экран `Home`:
  - заголовок;
  - счетчик нажатий;
  - кнопки `+1` и `Сброс`;
  - кнопка перехода на экран профиля;
- экран `Profile`:
  - текстовый блок;
  - кнопка `Назад`;
- навигационный граф со стартовым маршрутом `home`.

## Структура файлов

- `demo/MainActivity.kt` - входная точка приложения и подключение корневого composable;
- `demo/navigation/AppNavigation.kt` - маршруты `Home/Profile` и настройка `NavHost`;
- `demo/screens/HomeScreen.kt` - UI главного экрана и логика счетчика;
- `demo/screens/ProfileScreen.kt` - UI второго экрана и возврат назад.

## Требования для запуска

- Android Studio;
- проект с включенным Jetpack Compose;
- зависимости для Material 3 и Navigation Compose.

## Быстрый запуск

1. Создайте новый Android-проект с шаблоном Compose.
2. Проверьте пакет:
   - либо используйте `com.example.lesson01`;
   - либо поменяйте `package` и `import` в файлах демо под ваш пакет.
3. Скопируйте содержимое `lesson_01/demo` в исходники модуля `app`.
4. Запустите приложение на эмуляторе/устройстве.

## Сценарий проверки (взаимодействие)

1. Откройте приложение - должен появиться экран `Home`.
2. Нажмите `+1` несколько раз - число `Нажатий` увеличивается.
3. Нажмите `Сброс` - счетчик возвращается к `0`.
4. Нажмите `Перейти в профиль` - открывается экран `Профиль`.
5. Нажмите `Назад` - возврат на экран `Home`.

## Как устроено Android-приложение

### 1) Основные компоненты Android

- `Activity` - экран приложения и точка взаимодействия с пользователем.
- `Service` - фоновая работа без интерфейса (например, загрузка/синхронизация).
- `BroadcastReceiver` - реакция на системные события (интернет, зарядка, загрузка устройства).
- `ContentProvider` - безопасный доступ к данным приложения для других приложений.

### 2) Жизненный цикл `Activity`

Ключевые методы:

- `onCreate()` - создание экрана и инициализация;
- `onStart()` - экран становится видимым;
- `onResume()` - экран в фокусе, пользователь взаимодействует;
- `onPause()` - экран частично перекрыт;
- `onStop()` - экран не виден;
- `onDestroy()` - экран уничтожается.

Понимание жизненного цикла важно для корректной загрузки данных, освобождения ресурсов и стабильной работы UI.

### 3) UI-слой в Jetpack Compose

- Интерфейс описывается функциями с аннотацией `@Composable`.
- Экран собирается из компонентов: `Text`, `Button`, `Column`, `Row`, `Image` и т.д.
- Состояние (`State`) управляет отображением UI: когда состояние меняется, Compose перерисовывает зависимые части экрана.

Базовые инструменты состояния:

- `remember` - запоминает значение между перерисовками;
- `mutableStateOf` - создаёт наблюдаемое состояние.

### 4) Навигация между экранами

- `NavController` управляет переходами;
- `NavHost` описывает граф экранов;
- `composable("route") { ... }` связывает маршрут и конкретный экран.

Обычно архитектура простая: один `NavHost` в корневом экране + отдельные composable-функции для каждого экрана.

### 5) Структура проекта и роли файлов

- `AndroidManifest.xml` - объявление компонентов приложения, разрешений, intent-фильтров;
- `build.gradle(.kts)` - зависимости, версии SDK, параметры сборки;
- `res/values/` - строки, темы, цвета, размеры;
- `res/drawable/`, `res/mipmap/` - графические ресурсы и иконки.
